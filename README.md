# 今天吃什么

一个手机优先的网页小应用：输入地址后，查询附近 500 米内的餐厅和甜品下午茶，再随机抽一家去吃。

这是一个 v1 试手项目，重点是跑通从本地页面、地图接口代理到 Vercel 上线的完整链路。

## 本地使用

直接打开 `index.html` 可以体验界面。用 `file://` 打开时不会请求真实地图服务，会显示演示数据。

如需本地连接真实接口，需要自己准备本地服务代理 `/api/amap`，或直接部署到 Vercel 后测试。

## Vercel 部署

真实餐厅数据通过 Vercel Serverless Function 代理请求高德开放平台，避免把 Key 暴露在前端代码里。

需要在 Vercel 项目环境变量里配置：

- `AMAP_KEY`：高德开放平台「Web服务」Key，不是 Web端 JS API Key。
- 环境范围需要包含 `Production`。
- 修改环境变量后，必须重新 Redeploy 最新 Production，旧部署不会自动读取新值。

部署后可以用下面地址快速测试 Key 是否可用：

```text
https://你的域名/api/amap?action=geocode&address=深圳南头古城
```

正常返回里应该包含：

```json
"status": "1"
```

## 高德额度说明

v1 已经关闭自动预加载，只有用户输入地址建议、选择地址或点击搜索时才会请求地图服务，避免无感消耗调用量。

如果接口返回：

```text
SERVICE_NOT_AVAILABLE / infocode 10002
```

通常不是前端代码错误，而是高德账号、Key、免费额度、服务授权或计费状态导致服务不可用。可以等次日恢复，或在高德控制台确认 Web 服务 Key、账号余额和服务授权状态。

## 当前范围

- 地址联想：高德输入提示 API。
- 地址解析：高德地理编码 API。
- 附近餐饮：高德周边搜索 API。
- 随机推荐：前端本地随机选择。

后续如果继续优化，优先考虑进一步减少地图请求量，而不是继续扩功能。
